Question 4

If a and b are logical constants (one of True or False), not (a or b) is always the same as which of the following?

  • a or b
  • a and b
  • not a or not b
  • not a and not b

This is one of De Morganâ€™s laws. NOT (A OR B) is the same as NOT A AND NOT B: \neg \left(A \lor B\right) \quad\Leftrightarrow \quad \left(\neg A \right)\land \left(\neg B\right)

where \neg means NOT, \lor means OR and \land means AND. So the answer is not a and not b.

To see this in Python we can create two lists containing every possible combination of True and False:

a = [True, True, False, False]
b = [True, False, True, False]

NOT (A OR B) gives for each pair:

[not (a[i] or b[i]) for i in range(len(a))]
[False, False, False, True]

NOT A AND B gives for each pair:

[not a[i] and not b[i] for i in range(len(a))]
[False, False, False, True]

We can see that these are the same.

The other De Morganâ€™s Law is that NOT (A AND B) is the same as NOT A OR NOT B:

\neg \left(A \land B\right) \quad\Leftrightarrow \quad \left(\neg A \right)\lor \left(\neg B\right) Try write a set of Python commands to check this law as well!

Question 11

Count how many times the number 10 appears in the following list using 3 different methods:

  • Using a for loop.
  • Using a list comprehension
  • Using one of the listâ€™s methods.

You may find it useful to use the fact that True equals one and False equals zero when converted to an integer.

a = [20, 10, 10, 20, 10, 20, 10, 10, 20, 10]

The for loop approach:

counter = 0
for i in a:
    counter += i == 10
counter
6

The list comprehension approach:

sum([i == 10 for i in a])
6

The list method approach:

a.count(10)
6
